Changelog:
  1.1.0 — Stack-file auto-generation option
           * Added options.generate_stack (boolean, default false) to base JSON schema
           * When enabled with stages defined, Chopper emits one <stage>.stack per resolved stage
             alongside <stage>.tcl using the N/J/L/D/I/O/R format
           * Authoring guide §2.1 documents the derivation rules (dependencies > load_from > bare D)
           * example 03 (base_stages_only) and example 07 (base_full) demonstrate the new option
           * Feature is newly implemented; real-domain feedback and bug reports actively solicited
  1.0.2 — Agent workflow and deployment readiness update
           * Clarified collaborative JSON authoring workflow and user-in-the-loop analysis expectations
           * Added explicit domain-boundary confirmation rules and prohibition on scanning outside user-confirmed scope
           * Added runtime environment guidance: prefer Unix tcsh/csh analysis, then bash, then Windows-based fallback
           * Hardened validate_jsons.py for non-object JSON files and verified setup/validator smoke tests
  1.0.1 — Documentation and examples alignment update
           * Standardized JSON naming scheme to jsons/base.json and jsons/features/<feature_name>.feature.json
           * Updated examples and project path references to new jsons layout
           * Expanded analyzer instructions for proc call tracing logs, interactive feedback checkpoints, and EDA command/proc disambiguation
  1.0.0 — Initial release
           * depends_on field added to feature schema (array-only, minItems:1)
           * stage dependencies simplified to array-only in base and feature schemas
           * All stage field descriptions grounded in stack file N/J/L/D/I/O semantics
           * base-v1 structural repairs: load_from, steps, name field, stages block
