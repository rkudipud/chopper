R2G Tool Selection Guide

FlowBuilder VS Chopper

Both tools manage R2G flows, but they operate at fundamentally different levels. Match your domain to the right tool.

🔧
FlowBuilder
Stitch files into run scripts
Production
Pipeline
flow.json + features → run_<stage>.tcl → SNORT → P4 template
📍 Domains
apr_fc rtlfp
File-organised flows with clear stage/step separation
🎯 Scope
File-level only. Adds or removes whole .tcl files.
No visibility inside files — unused procs and dead code still ship.
🧠 Run Script
✓ Full Semantic parsing.
Classifies every command: iproc_source, proc call, #if/#else, comment. Wires them correctly into output.
📦 Proc Aware
✗ None
Knows which files are sourced — not what procs they define or call.
📂 File Config
⚠ Manual Two places. Configure both flow JSON (run script) and SNORT config (P4 filter) separately.
🐍 Python
✓ Supported Stages and features can be Python-based.
🎯 Choose FlowBuilder when…
  • Your domain is apr_fc or rtlfp
  • Flow uses separate stage/step files sourced together
  • You need to add, remove, or reorder steps via JSON config
  • You have Python-based stages or features
  • You do not need to prune unused procs from inside files
✂️
Chopper
Trim domains to what you actually need
In Development
Pipeline
base.json + features + project.json → analyze → trim
📍 Domains
sta_pt fev_formality power lvs_icv …
All signoff and analysis tool-flow domains
🎯 Scope
File-level + proc-level. Parses inside Tcl, resolves proc definitions, and traces transitive calls.
Removes exactly what is unused — nothing more, nothing less.
🧠 Run Script
⚠ String-based F3 feature only.
Writes steps as-is — does not classify commands semantically. For intelligent run-script construction, use FlowBuilder.
📦 Proc Aware
✓ Full
Indexes every proc, follows iproc_source chains, and builds the complete transitive call graph before trimming.
📂 File Config
✓ Native One place.
files.include / files.exclude are first-class. No separate tool config needed.
🐍 Python
✗ Not supported Tcl-only. No Python, Perl, or csh parsing.
🎯 Choose Chopper when…
  • Your domain is a signoff or analysis tool (STA, FEV, Power, LVS…)
  • Domain is large and bloated — ships far more than you need
  • You need to remove unused Tcl procs from inside shared library files
  • You have files that must survive trimming even if not in any run script
  • You want a single JSON config — no separate SNORT setup
